# Contributing to claude-code-prompt

Thank you for your interest in contributing! This project documents Claude Code's prompt engineering system, and community contributions help keep it accurate and comprehensive.

## How to Contribute

### Reporting Issues

- **Inaccuracies**: If you notice prompt content that doesn't match the current version of Claude Code, please open an issue with:
  - The specific document and section
  - What you believe is incorrect
  - The correct information (if known)

- **Missing Content**: If you discover prompts or features not yet documented, open an issue describing what's missing.

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b fix/update-bash-prompt`)
3. **Make your changes** following the style guidelines below
4. **Commit** with a clear, descriptive message in Chinese (project convention)
5. **Push** and open a **Pull Request**

### Style Guidelines

- **Language**: Document content should be in Chinese, with original English prompt text preserved in code blocks
- **Mermaid Diagrams**: Use Mermaid for architecture and flow diagrams wherever possible
- **File Structure**: Follow the existing numbered file naming convention (`00-`, `01-`, etc.)
- **Prompt Quoting**: Always quote original prompt text in fenced code blocks (` ``` `)
- **Source References**: Include source file paths when documenting prompts

### Document Structure

Each document should follow this structure:

```markdown
# Title

> Source file reference and brief description

---

## Section with Mermaid diagram (if applicable)

## Prompt content in code blocks

## Notes and analysis
```

## Code of Conduct

- Be respectful and constructive
- Focus on accuracy and clarity
- Credit original sources appropriately

## Questions?

Open an issue if you have questions about contributing.
